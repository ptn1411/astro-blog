# Top 12 công cụ debugging tốt nhất 2025 – Dev nào cũng phải biết

Năm 2025, việc “console.log + đoán mò” đã chính thức bị khai trừ khỏi danh sách kỹ năng lập trình viên chuyên nghiệp.

Dưới đây là **12 công cụ debugging đang được dùng nhiều nhất** (mình và team đang dùng hàng ngày), xếp theo thứ tự từ miễn phí → trả phí, từ cơ bản → thần thánh.

## 1. Chrome DevTools (vẫn là vua – miễn phí)

Nâng cấp 2025:

- Tab **Recorder** + export thành Playwright/Cypress test tự động
- **CSS Overview** + **Lighthouse CI** tích hợp ngay trong DevTools
- Performance panel có thể zoom từng frame 1ms

Mẹo nhanh: Nhấn `Ctrl + Shift + P` → gõ “record” → quay lại toàn bộ hành động người dùng → export code test ngay lập tức.

## 2. React DevTools + Redux DevTools (miễn phí)

Phiên bản 2025 đã hỗ trợ:

- React Server Components
- Xem diff state theo từng commit
- Time-travel debugging cho Redux

## 3. VS Code Debugger (miễn phí)

Breakpoints có điều kiện + logpoints (in log mà không dừng chương trình) → thần thánh khi debug vòng lặp lớn.

## 4. Cursor / GitHub Copilot X Debugging (AI – miễn phí + trả phí)

Cách dùng 2025:

1. Highlight đoạn code lỗi
2. Nhấn `Cmd + L` (Cursor) hoặc `Copilot Chat`
3. Gõ: “Explain why this crashes” → nhận giải thích + cách fix trong 5 giây

## 5. Sentry (miễn phí đến 5k events/tháng)

Tính năng 2025 đáng tiền nhất:

- Source map tự động
- Session replay (xem lại video người dùng thực hiện trước khi crash)
- AI gợi ý nguyên nhân lỗi ngay trong dashboard

## 6. LogRocket (trả phí – từ $99/tháng)

“Google Analytics cho UX”:

- Quay lại toàn bộ phiên người dùng (video thật)
- Xem lại Redux state, network, console.log từng frame
- Tự động phát hiện “rage click” và ANR

## 7. Replay.io (mới nổi cực mạnh 2025)

Ghi lại toàn bộ runtime → bạn có thể **debug quá khứ**:

- Thêm console.log vào quá khứ mà không cần chạy lại app
- Di chuyển thời gian như xem YouTube

Miễn phí cho dự án open-source, trả phí cho team.

## 8. Playwright Test + Trace Viewer

Chạy test thất bại → mở Trace Viewer → xem:

- Screenshot từng bước
- Network waterfall
- Console log + source code tại đúng thời điểm lỗi

## 9. Datadog RUM & APM

Dành cho team lớn, muốn biết:

- Lỗi frontend ảnh hưởng bao nhiêu người dùng thực tế
- API nào chậm → gây lỗi timeout ở client

## 10. BrowserStack / Sauce Labs

Debug trên 3000+ thiết bị thật cùng lúc (iPhone cũ, Android rẻ tiền… là nơi bug hay ẩn nấp nhất).

## 11. WhyDidYouRender (React)

Thêm 1 dòng → biết ngay component nào re-render không cần thiết.

```plain
npm install @welldone-software/why-did-you-render
```

```plain
if (process.env.NODE_ENV === 'development') {
  const whyDidYouRender = require('@welldone-software/why-did-you-render');
  whyDidYouRender(React, { trackAllPureComponents: true });
}
```

## 12.DebugBear (mới toanh 2025)

Chuyên đo performance + tự động phát hiện regression:

- Core Web Vitals giảm → báo ngay
- Tự động so sánh trước/sau deploy

**Bảng so sánh nhanh (2025)**

| Công cụ | Miễn phí? | Quay video session | Time-travel | AI gợi ý fix | Tốt nhất cho |
| --- | --- | --- | --- | --- | --- |
| Chrome DevTools | Có | Không | Không | Không | Mọi thứ cơ bản |
| Sentry | Có (giới hạn) | Có (add-on) | Không | Có | Production errors |
| LogRocket | Không | Có | Không | Có | UX + frontend bug |
| Replay.io | Có (open-source) | Có | Có | Có | Debug “khó nhai” |
| Cursor/Copilot | Có + trả phí | Không | Không | Có | Fix nhanh khi code |

## Kết luận

Năm 2025, dev giỏi không phải là người ít tạo bug nhất, mà là **người tìm và sửa bug nhanh nhất**.Hãy chọn ít nhất 3 công cụ trong danh sách trên:

1. Chrome DevTools (bắt buộc)
2. Cursor hoặc Copilot (hằng ngày)
3. Sentry hoặc LogRocket (production)
